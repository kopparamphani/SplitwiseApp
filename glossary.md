# Glossary — Plain Words for Tech Terms

A living dictionary for this project. Caveman rules: few words, simple, with a quick "Like:" picture.
Add new terms as we hit them. Keep it honest — if a word here stops being true, fix it.

---

### Agent
A helper that does one job on its own. You tell it the goal, it figures out the steps.
**Like:** a worker who knows their trade. You say "build wall," they grab bricks themselves.

### Agent Orchestration
Many agents working together, one telling the others when to go.
**Like:** a foreman pointing — "you dig, then you pour, then you paint."

### API (Application Programming Interface)
A door one program uses to ask another program for something.
**Like:** a restaurant window. You order, kitchen cooks, food comes back. You never enter the kitchen.

### API Testing
Checking that door works right — ask it things, see if the answer is correct.
**Like:** shouting an order through the window and making sure the right food comes out.

### Architecture Diagram
A picture of all the parts and how they connect.
**Like:** a map of the village — huts, roads, well, where each thing sits.

### Argo CD
A robot that watches your "what should be running" notes and makes the real system match them.
**Like:** a tidy roommate who keeps the room exactly like the photo on the wall.

### Automation Testing
Tests that run by themselves, no human clicking.
**Like:** a trap you set once that checks the cave every night while you sleep.

### Branch
A side copy of the code where you try changes without breaking the main one.
**Like:** a practice scroll. Scribble freely, copy to the real book only when it's good.

### CI/CD (Continuous Integration / Continuous Delivery)
A pipeline that auto-checks new code and auto-ships it when it passes.
**Like:** a conveyor belt — drop in raw meat, it gets inspected, cooked, and served, no hands.

### C4 Diagram
A way to draw a system at four zoom levels: whole thing → big parts → inner parts → code.
**Like:** map of the world → country → city → street. Zoom in step by step.

### Commit
Saving a chunk of work with a note saying what you changed.
**Like:** stamping the date in a logbook: "today I added the fire pit."

### Container
A sealed box holding your app plus everything it needs to run.
**Like:** a packed lunchbox. Same meal anywhere — home, work, mountain.

### Deployment
Putting your app where people can actually use it.
**Like:** opening the shop doors for customers.

### Docker
The tool that makes and runs containers (the lunchboxes).
**Like:** the machine that packs the lunchbox the same way every time.

### Docs-as-Code
Writing documents the same way you write code — in text files, version-tracked, reviewed.
**Like:** keeping your notes in the same logbook as your work, not on loose leaves.

### Environment (Local / Production)
The place where code runs. **Local** = your own machine. **Production** = the real, live one people use.
**Like:** Local is your backyard practice. Production is the real hunt.

### GitHub
A website where code lives, gets shared, and gets reviewed by a team.
**Like:** the shared cave wall where everyone keeps and checks the tribe's drawings.

### Helm Chart
A bundle of recipes that installs a whole app onto Kubernetes in one go.
**Like:** a meal kit — all ingredients and instructions in one bag.

### Integration Test
Checking that two or more parts work right *together*, not just alone.
**Like:** testing that the bow AND arrow work as a pair, not just each by itself.

### Kubernetes (often "K8s")
A boss that runs many containers, restarts dead ones, and adds more when busy.
**Like:** a herd master — keeps every animal alive, fed, and replaces any that fall.

### MCP (Model Context Protocol)
A standard plug that lets an agent safely use outside tools and data.
**Like:** a universal socket — any tool plugs in the same way.

### Microservice
One small app that does one job, talking to other small apps.
**Like:** one specialist in the tribe — the toolmaker, the cook, the healer. Each does their thing.

### Monolith (the opposite, for contrast)
One big app that does everything in a single lump.
**Like:** one giant hut where cooking, sleeping, and storage all share one room.

### Observability
Being able to see what your system is doing and why, from the outside.
**Like:** smoke signals and tracks — you read them to know what's happening in the forest.

### Production-Grade
Built solid enough for real people and real use, not just a toy.
**Like:** a real bridge that holds weight, not a stick across a stream.

### Pull Request (PR)
Asking the team to review and accept your branch changes into the main code.
**Like:** showing your practice scroll to the elders before it goes in the real book.

### Repository (Repo)
The folder/home where all of a project's code lives.
**Like:** the chest where the whole tribe keeps its scrolls.

### SDLC (Software Development Life Cycle)
The full journey of building software: idea → design → build → test → ship → maintain.
**Like:** the seasons of a crop — plant, grow, harvest, store, repeat.

### Unit Test (UT)
Checking one tiny piece of code works alone.
**Like:** testing one arrowhead is sharp before you build the arrow.

### YAML
A simple text format for writing settings and instructions, easy for humans to read.
**Like:** a tidy shopping list — one item per line, spacing shows what belongs to what.

---

*Style note: every entry stays short. If you can't explain it in a breath, it's not caveman enough — flag it and we'll simplify.*
